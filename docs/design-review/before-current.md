# Before Current Visual Redesign

## Problems Observed

- Homepage looked like an engineering form instead of a product entry.
- Ordinary users saw provider/model/source-data language before the report value.
- Input, model setup, report content, source data, export, and disclaimer were stacked into one long surface.
- Report sections overemphasized raw paipan and source traces, while users wanted dimension scores, dayun curves, color scales, and tables.
- English labels such as Provider, Report Status, Dimension Spectrum, Birth Input, and Raw JSON leaked into the product UI.
- Mobile layout relied on long tables and dense sections without enough internal scrolling.

## Visual Baseline

The baseline was captured mentally from the second-round public demo state and documented here before applying `visual-report-redesign`. The new screenshot set is stored in `docs/design-review/screenshots/`.
